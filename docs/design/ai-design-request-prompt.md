# Commit Grow 디자인 시스템 + 화면 목업 요청

## 서비스 개요

Commit Grow는 GitHub 활동(commit, PR, issue, 코드리뷰)을 자동 수집·집계하고, AI가 회고(KPT: Keep/Problem/Try) 작성을 돕는 개발자 성장 플랫폼입니다. 스택은 Next.js(App Router) + Tailwind + shadcn/ui.

전체 구현이 아니라 **디자인 시스템 정의 + 주요 화면 목업**만 필요합니다. 컴포넌트 코드 스캐폴딩은 별도로 처리하므로, 시간을 여기에 쓰지 말고 아래 두 가지에 집중해주세요.

1. Tailwind 설정값 + shadcn 컴포넌트를 커스터마이징한 Commit Grow 전용 디자인 토큰(색상, 타이포, radius, spacing 등)
2. 아래 화면 정의를 기준으로 한 주요 화면 목업 (Success / Empty / Loading 상태 포함)

## 핵심 기능 흐름

- GitHub OAuth로 로그인, 최초 로그인 시 계정 생성
- 로그인 후 대시보드 진입: 히트맵, 스트릭 배지, 액션포인트(TODO/FAILED/ROUTINE) 목록, 회고 요약 카드 노출
- "오늘의 회고 작성" 시작 → 기존 액션포인트 리뷰(완료/실패/루틴/철회 선택) → 오늘의 Git 활동 확인(활동 없으면 직접 입력) → AI가 활동 기반 질문 최대 3개 생성
- 질문에 하나씩 답변하는 단계형 폼(진행률 표시, 예: 1/3) → 답변마다 AI가 K/P/T로 실시간 재구성·누적
- 모든 질문 답변 후 KPT 전체를 검토·수정하여 확정 저장 → 이어서 Try 기반 AI 액션포인트 후보를 채택하거나 직접 추가, 목적(Purpose) 태그 연결
- 완료 시 대시보드로 복귀
- GNB에서 회고 기록 목록(검색 가능) → 회고 상세(질문/답변, KPT, AI 요약/인사이트, 연결된 액션포인트, 삭제 가능. 수정은 불가)
- 히트맵의 특정 날짜 클릭 시 활동 요약 + 회고 미리보기 팝업(모달)

## 대상 화면 정의

**S-01 로그인**
- 목적: GitHub OAuth 로그인 진입점
- 구성: 서비스 로고/이름, 한 줄 소개 문구, "GitHub으로 로그인" 버튼
- 상태: Loading(OAuth 리다이렉트 중 버튼 비활성)

**S-03 대시보드**
- 목적: 로그인 후 기본 진입점, 활동/회고 현황 파악 + 오늘의 회고 작성 유도
- 구성: ① 히트맵(최근 N주 활동) ② 스트릭 배지 ③ "오늘의 회고 작성하기" CTA 버튼 ④ 액션포인트 목록(TODO/FAILED/ROUTINE, 읽기 전용) ⑤ 회고 요약 카드 리스트(최근순, 분석중 상태 태그 포함)
- 상태: Empty(신규 사용자 → 기초 가이드 + 첫 회고 유도), Loading(위젯별 스켈레톤), Error(위젯 단위)

**S-04 회고 작성 — 진입 (액션포인트 리뷰 / 활동 확인)**
- 목적: 회고 작성 전 기존 액션포인트 정리 + 오늘의 회고 소재 확보
- 구성: ① TODO/FAILED/ROUTINE 액션포인트 목록(항목별 완료/실패/루틴/철회 토글, 목록 없으면 섹션 숨김) ② Git 활동 요약 또는 직접 입력 텍스트 필드 ③ "다음" 버튼
- 상태: Loading(질문 생성 대기 중 버튼 비활성), Empty(활동 없음 → 직접 입력 UI 전환)

**S-05 회고 작성 — 질문 답변 (단계형 폼)**
- 목적: 질문에 하나씩 답변하며 K/P/T 실시간 누적
- 구성: ① 진행률 표시("1/3") ② 현재 질문 텍스트 ③ 답변 입력 필드(서술형) ④ "다음" 버튼 ⑤ (선택) 누적 K/P/T 미리보기 사이드 영역
- 상태: Loading(재구성 대기 중), Error(인라인 에러)
- 비고: 챗봇형 대화 UI가 아니라 회원가입 마법사형 단일 질문 폼

**S-06 회고 작성 — 최종 확인 (KPT 확정 → 액션포인트 작성)**
- 목적: ① 누적된 K/P/T 검토·수정 후 확정 저장 ② 이어서 액션포인트 채택/추가 + 목적(Purpose) 연결
- 구성 ①: Keep/Problem/Try 목록(편집 가능, Try 최대 6개), 항목 추가 버튼, "저장" 버튼
- 구성 ②: 액션포인트 후보 목록(AI 자동 생성), 직접 추가 버튼, 목적 선택/입력 UI(드롭다운 + 새 목적 입력, 태그 표시 + 수정 버튼), "완료"/"건너뛰기" 버튼
- 상태: Loading(저장/생성 요청 중), 개별 항목 인라인 로딩

**S-07 회고 기록 — 목록**
- 목적: 저장된 회고를 시간순 조회·검색
- 구성: ① 검색 입력 ② 회고 카드 리스트(날짜, 요약 일부, 분석 상태 태그) ③ 페이지네이션
- 상태: Empty(회고 없음 → 첫 회고 유도), Loading(스켈레톤)

**S-08 회고 상세**
- 목적: 개별 회고 전체 내용 열람 및 삭제
- 구성: ① 회고 날짜 ② 질문/답변 전체 ③ K/P/T ④ AI 요약/인사이트(또는 "분석중") ⑤ 연결된 액션포인트 목록 ⑥ "삭제" 버튼
- 비고: 수정 버튼 없음(회고는 수정 불가)

**S-09 히트맵 상세 팝업**
- 목적: 특정 날짜의 활동과 회고 요약 빠른 미리보기
- 구성: ① 날짜 ② 활동 요약(커밋/PR 수 등) ③ 회고 요약 미리보기 + "상세 보기" 링크 ④ 회고 없을 시 안내 문구
- 형태: 모달(라우트 이동 없음)
