# [2단계] 화면별 생성 프롬프트

1단계에서 만든 스타일 가이드(포인트 #18F293 / 배경 #FFFFFF / 텍스트 #3E3E40 / 구분영역 #FBFBFC / 카드 #FEFEFE, Pretendard, H1 24px·700 / H2 18px·600 / Body 14px·400 / Caption 12px·400)와 플랫폼 조건(데스크톱 웹 전용, 기준 폭 1440px, 반응형 대응 없음)을 그대로 유지한 채로, 화면을 하나씩 순서대로 생성해줘.

화면마다 아래 문구를 프롬프트 맨 앞에 반복해서 넣어줘: "데스크톱 웹 1440px 기준, 반응형 없음."

무료 티어는 캔버스당 화면 수 제한이 있을 수 있으니, 아래를 5개 이하 단위로 나눠서 진행해줘(예: S-01~S-05 먼저, 이후 S-06~S-09).

---

**S-01 로그인**
1단계 스타일 + 데스크톱 웹 1440px 기준 유지. 구성: 서비스 로고/이름, 한 줄 소개 문구, "GitHub으로 로그인" 버튼. 상태: Loading(OAuth 리다이렉트 중 버튼 비활성).

---

**S-03 대시보드**
1단계 스타일 + 데스크톱 웹 1440px 기준 유지. 구성: ① 히트맵(최근 N주 활동) ② 스트릭 배지 ③ "오늘의 회고 작성하기" CTA 버튼 ④ 액션포인트 목록(TODO/FAILED/ROUTINE, 읽기 전용) ⑤ 회고 요약 카드 리스트(최근순, 분석중 상태 태그). 상태: Empty(신규 사용자용 가이드 문구), Loading(위젯별 스켈레톤).

---

**S-04 회고 작성 — 진입**
1단계 스타일 + 데스크톱 웹 1440px 기준 유지. 구성: ① 액션포인트 목록(항목별 완료/실패/루틴/철회 토글) ② Git 활동 요약 또는 직접 입력 텍스트 필드 ③ "다음" 버튼. 상태: Loading(버튼 비활성), Empty(직접 입력 UI로 전환).

---

**S-05 회고 작성 — 질문 답변**
1단계 스타일 + 데스크톱 웹 1440px 기준 유지. 회원가입 마법사형 단일 질문 폼(챗봇 UI 아님). 구성: ① 진행률 표시("1/3") ② 질문 텍스트 ③ 답변 입력 필드 ④ "다음" 버튼 ⑤ (선택) 누적 K/P/T 미리보기. 상태: Loading, Error(인라인).

---

**S-06 회고 작성 — 최종 확인**
1단계 스타일 + 데스크톱 웹 1440px 기준 유지. 구성 ①: Keep/Problem/Try 목록(편집 가능, Try 최대 6개), 항목 추가 버튼, "저장" 버튼. 구성 ②(①저장 후 노출): 액션포인트 후보 목록, 직접 추가 버튼, 목적(Purpose) 선택/입력 UI(드롭다운+새 목적 입력, 태그+수정 버튼), "완료"/"건너뛰기" 버튼.

---

**S-07 회고 기록 — 목록**
1단계 스타일 + 데스크톱 웹 1440px 기준 유지. 구성: ① 검색 입력 ② 회고 카드 리스트(날짜, 요약 일부, 분석 상태 태그) ③ 페이지네이션. 상태: Empty(첫 회고 유도), Loading(스켈레톤).

---

**S-08 회고 상세**
1단계 스타일 + 데스크톱 웹 1440px 기준 유지. 구성: ① 회고 날짜 ② 질문/답변 전체 ③ K/P/T ④ AI 요약/인사이트(또는 "분석중") ⑤ 연결된 액션포인트 목록 ⑥ "삭제" 버튼. 수정 버튼은 없음.

---

**S-09 히트맵 상세 팝업**
1단계 스타일 + 데스크톱 웹 1440px 기준 유지. 모달 형태(라우트 이동 없음). 구성: ① 날짜 ② 활동 요약(커밋/PR 수 등) ③ 회고 요약 미리보기 + "상세 보기" 링크 ④ 회고 없을 시 안내 문구.
