# 디자인 목업 작업 방향 (재정리)

## 확정된 사실

- Claude Design(claude.ai/design)은 Anthropic Labs가 만든 별도 제품. `/design-sync`를 실행해도 로컬 수정이 자동 반영되지 않음(직접 확인) — 동기화는 **수동**: 매번 수정 내용을 Claude Design 프로젝트에 다시 첨부해야 반영됨
- 지금 commit-grow는 프론트엔드 코드가 아직 없어서(백엔드만 존재) "코드베이스에서 디자인 시스템 추출" 경로는 못 씀 — 대신 문서/에셋 업로드로 부트스트랩해야 함
- Claude Design은 구조·디자인 시스템 일관성 관리에는 유리하지만 비주얼 완성도는 떨어짐 — 구조/톤 기준은 Claude Design에서 잡고, 최종 비주얼은 v0·Stitch 등을 함께 써서 뽑는 방식으로 병행

## 진행 순서

### 0단계. 준비물 확정 (완료)
- **플랫폼: 데스크톱 웹 전용, 기준 화면 폭 1440px (반응형 대응 없음)** — 명세서에 없던 항목이라 누락돼 있었음, 확정함
- 디자인 컨셉: 색상(#18F293/#FFFFFF/#3E3E40/#FBFBFC/#FEFEFE), Pretendard, 타입 스케일(H1~Caption)
- 화면정의서 8개 화면(S-01, S-03~S-09) 요약
- → `docs/design/ai-design-request-prompt.md`, `01-style.md`, `02-screens.md`에 이미 정리됨(플랫폼 조건 반영 완료)

### 1단계. Claude Design 디자인 시스템 최초 세팅 (1회만)
1. claude.ai/design 접속 → 조직 생성/선택 → 온보딩 진입
2. 코드베이스가 없으므로 "문서/개별 에셋" 경로로 업로드:
   - `01-style.md` (색상/폰트/타입 스케일)
   - `01-feature-spec.md`, `04-screen-spec.md`
3. Claude가 추출한 디자인 시스템(컬러 팔레트, 타이포, 컴포넌트, 레이아웃 패턴)이 지정값(#18F293 등) 및 플랫폼 조건(데스크톱 웹 1440px)과 정확히 일치하는지 확인 — 틀리면 "Remix"로 직접 수정 요청
4. 테스트 프롬프트("대시보드 화면 만들어줘" 등)로 검증 후, 만족스러우면 "Published" 토글 켜기

**주의**: 이 단계에서 색상/폰트가 어긋나면 이후 모든 화면에 그대로 전파됨. 반드시 여기서 값 대조부터 하고 넘어갈 것.

### 2단계. 화면별 순차 생성
- 한 번에 다 요청하지 말고 5개 이하로 배치: S-01~S-05 먼저, 이후 S-06~S-09
- 화면마다 `02-screens.md`에 정리된 구성요소/상태를 그대로 프롬프트에 포함
- 생성 직후 화면정의서와 항목 대조(구성요소 누락, 상태 케이스, 인터랙션) — 여기서 놓치면 이후 단계까지 그대로 감

### 3단계. 톤 일관성 검증
- 화면 8개 다 나오면 육안이 아니라 실제 색상 hex/폰트/radius 값을 뽑아 비교
- 화면 폭도 함께 확인 — 1440px 기준에서 벗어나거나 화면마다 폭이 다르면 반드시 재요청
- 어긋나는 화면이 있으면 "1단계에서 확정한 디자인 시스템 기준으로 다시 맞춰줘"라고 재요청

### 4단계. 명세서 변경 시 갱신 루틴 (자동 동기화 없음 전제)
1. 기능명세서/화면정의서 수정
2. 변경이 어느 화면에 영향 주는지 확인 → 해당 화면만 `02-screens.md`에서 수정
3. Claude Design 프로젝트 채팅에 수정된 내용을 다시 첨부하거나 설명 → 그 화면만 재생성
4. 디자인 시스템 자체(색상/폰트 등)가 바뀌는 경우만 조직 설정 → "Remix"로 시스템 자체를 업데이트하고 전체 화면 재검증

### 4-1단계. 비주얼 마무리 (v0 / Stitch 등)
- Claude Design에서 확정한 화면 구조 + 1단계 디자인 토큰(플랫폼 조건 포함: 데스크톱 웹 1440px)을 그대로 v0나 Stitch에 넘겨서 최종 비주얼 완성도를 올림
- 이때도 화면을 한 번에 다 넣지 말고 5개 이하로 나눠서, 토큰 값을 매번 반복 명시(앞서 겪은 톤 불일치 재발 방지)
- 완성된 비주얼이 Claude Design 구조와 어긋나지 않는지(구성요소 누락 등) 한 번 더 대조

### 5단계. 코드 반영
- 화면들이 확정되면 `/design-sync`(Claude Code)로 handoff → Next.js + Tailwind + shadcn 프로젝트 스캐폴딩 시작
- 이 시점부터는 코드가 소스 오브 트루스. 이후 디자인을 더 고치고 싶으면 코드 우선 수정 후 필요 시에만 Design 쪽에 재반영(양방향이지만 수동)

## 툴별 역할 분담
- **Claude Design**: 화면 구조, 구성요소·상태 누락 검증, 디자인 시스템(토큰) 일관성 관리
- **v0 / Stitch 등**: 최종 비주얼 완성도 — Claude Design에서 나온 구조를 그대로 입력값으로 받아 비주얼만 다듬음
- 툴을 오갈 때는 항상 같은 디자인 토큰 값을 반복 명시해서 톤 불일치를 방지
