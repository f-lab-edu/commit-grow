## 부록: 확정되지 않은 아이디어

아래는 검토했지만 도입 여부를 확정하지 않은 아이디어다. 본 기획서의 기능명세·화면정의서·API 명세에는 반영하지 않았다.

### 더글러스 엥겔바트 A/B/C 작업 구분 모델

- A: 단순 수행, B: 일하는 방식 개선, C: 개선하는 방식을 개선
- 논의된 활용 방향: 회고 내용을 A/B/C로 분류해 회고의 "깊이"를 분석하거나, 장기 성장 지표로 활용
- 미확정 사유: 분류 기준(어떤 회고 내용이 B/C에 해당하는지)이 아직 정의되지 않았고, 초기 데이터가 적은 단계에서 정교한 지표를 도입하는 것은 [서비스 개요의 초기 단계 성공 기준](./00-overview.md#6-초기-단계-성공-기준)과도 우선순위가 맞지 않음
- 재논의 시점: 회고 작성 루프가 안정화되고 사용자 데이터가 축적된 이후
