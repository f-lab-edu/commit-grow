## FT-04. 액션포인트 리뷰 (회고 작성 진입)

| 항목 | 내용 |
| --- | --- |
| 설명 | 기존 TODO·FAILED·ROUTINE 상태 액션포인트가 있으면 항목별로 "완료" / "실패" / "루틴" / "철회" 중 하나를 선택받는다. 선택 즉시 서버에 반영된다(절차저장형). 완료를 선택하면 상태를 `DONE`으로 변경해 더 이상 노출하지 않는다. 실패를 선택하면 다음에 다시 수행하는 것으로 보아 상태를 `FAILED`로 변경하고(`failureCount` +1) 다음 회고 진입 시에도 다시 노출된다. 루틴을 선택하면 이미 수행했지만 성격상 다음에도 반복 수행이 필요한 항목으로 보아 상태를 `ROUTINE`으로 변경하고 다음 회고 진입 시에도 계속 노출된다. 철회를 선택하면 불필요해진 항목으로 보아 상태를 `REMOVED`로 변경해 더 이상 노출하지 않는다 |
| 트리거 | 회고 작성 진입 화면 로드 |
| 관련 화면 | S-04 회고작성 - 진입 |
| 관련 API |  |
| 비고 | 액션포인트가 하나도 없으면 이 단계는 스킵하고 바로 다음 단계로 진행. 액션포인트 상태 모델은 `TODO`/`DONE`/`FAILED`/`ROUTINE`/`REMOVED` 5분류로 확정(05번 참고). 현재 상태와 무관하게 4개 결정 모두 선택 가능(전이 제한 없음). 선택한 결정은 항목별로 즉시 저장되며, 더 이상 로컬 보관 후 일괄 반영하지 않음(과거 "FT-08 시점 일괄 반영" 방식에서 변경). 리뷰 화면과 대시보드는 `TODO`·`FAILED`·`ROUTINE` 상태를 모두 조회 대상으로 포함해야 함(복수 상태 필터, 구현 시 상세 결정) |

## FT-07. 최종 확인(KPT 저장) 및 액션포인트 작성

| 항목 | 내용 |
| --- | --- |
| 설명 | 2단계로 진행된다. **① KPT 최종 확인**: 모든 질문에 답변 후 누적된 K/P/T 전체를 검토하며 사용자가 직접 항목을 추가·수정·삭제할 수 있다. 확정 후 저장하면 이 시점에 Retrospect·RetrospectAnswer·KptItem이 서버에 즉시 생성된다(스트릭 +1, 비동기 요약/인사이트/제목 분석 job도 이 시점에 큐 적재). **② 액션포인트 작성**: KPT 저장 완료 후 이어서 진입. 확정된 Try 목록을 기반으로 AI가 액션포인트 후보를 생성해 미리보기로 노출하고, FT-04에서 "실패"/"루틴"으로 리뷰된 기존 항목을 참고한 파생 초안도 함께 노출한다. 사용자가 후보를 채택하거나 직접 새 항목을 추가하면 그 액션포인트는 개별적으로 즉시 서버에 저장된다(TODO 상태 생성, 절차저장형 — 더 이상 전체를 저장 시점에 일괄 생성하지 않음). 각 액션포인트는 "목적(Purpose)"과 연결할 수 있다 — 기존 활성 목적 중 선택하거나 새 목적을 입력해 추가하며, 목적 추가·수정은 액션포인트 저장과 별개로 그 즉시 저장된다. 연결된 목적은 태그 형태로 표시되고 태그의 수정 버튼으로 문구를 고칠 수 있다(수정 시 "잦은 목적 수정은 방향성을 해칠 수 있어요" 안내 문구 노출, 저장 후 같은 목적을 공유하는 다른 액션포인트의 표시도 함께 갱신됨) |
| 트리거 | 마지막 질문 답변 제출(① 진입) / KPT 저장 완료(② 진입) |
| 관련 화면 | S-06 회고작성 - 최종확인 |
| 관련 API | `POST /retrospects/action-points/generate` (액션포인트 후보 생성, 미리보기용 저장 없음). 그 외 저장 관련 API는 추후 확정 |
| 비고 | 활성(TODO+FAILED+ROUTINE) 액션포인트가 16개인 상태에서 새 항목을 추가하려 하면 그 항목만 저장이 생략되고 화면에 경고 문구가 표시된다(다른 항목 추가는 계속 가능, 별도 차단 UI 없음). 목적 드롭다운 후보는 현재 활성 액션포인트에 걸린 목적만 노출(완료된 목적은 자연히 후보에서 빠짐). 목적은 별도 관리 화면이 없고 이 화면(액션포인트 작성 모달)에서만 추가·수정 가능. FT-04에서 리뷰한 기존 액션포인트 원본(row)의 content는 여기서 수정하지 않으며, 내용을 조정하고 싶다면 참고해서 새 항목을 추가하는 방식으로 처리한다(원본과 FK 연결 없음, 맥락 추적은 sourceRetrospectId와 공유 목적으로 가능). 신규 후보 간, 신규-기존 항목 간 중복 체크는 하지 않음. 사용자 직접 추가 항목은 존댓말·표준어 보정 없이 입력 그대로 저장. Try 목록 자체 상한은 6개(기존 유지) |

## FT-08. 액션포인트 작성 완료 (위저드 종료)

| 항목 | 내용 |
| --- | --- |
| 설명 | 액션포인트 작성 단계에서 "완료"(또는 "건너뛰기") 클릭 시 위저드를 종료하고 대시보드로 이동한다. 액션포인트·목적은 FT-07 ②에서 이미 각각 즉시 저장되었고, 회고 본문(KPT)도 FT-07 ①에서 이미 저장되었으므로 이 시점에는 별도 저장 없이 화면 전환만 수행한다 |

| 관련 화면 | S-06 → S-03(대시보드 복귀) |
| 관련 API |  |
| 비고 | 스트릭 +1과 비동기 분석 job 적재는 FT-07 ①(KPT 저장) 시점에 이미 처리됨 — 액션포인트를 하나도 만들지 않아도 회고 자체는 완료로 인정된다. 대시보드에는 액션포인트 변경사항이 이미 반영된 상태로 진입하며, 요약 카드만 "분석중" 상태로 노출 |
