## FT-04. 액션포인트 리뷰 (회고 작성 진입)

| 항목 | 내용 |
| --- | --- |
| 설명 | 기존 TODO·FAILED·ROUTINE 상태 액션포인트가 있으면 항목별로 "완료" / "실패" / "루틴" / "철회" 중 하나를 선택받는다. 완료를 선택하면 상태를 `DONE`으로 변경해 더 이상 노출하지 않는다. 실패를 선택하면 다음에 다시 수행하는 것으로 보아 상태를 `FAILED`로 변경하고(`failureCount` +1) 다음 회고 진입 시에도 다시 노출된다. 루틴을 선택하면 이미 수행했지만 성격상 다음에도 반복 수행이 필요한 항목으로 보아 상태를 `ROUTINE`으로 변경하고 다음 회고 진입 시에도 계속 노출된다. 철회를 선택하면 불필요해진 항목으로 보아 상태를 `REMOVED`로 변경해 더 이상 노출하지 않는다 |
| 트리거 | 회고 작성 진입 화면 로드 |
| 관련 화면 | S-04 회고작성 - 진입 |
| 관련 API |  |
| 비고 | 액션포인트가 하나도 없으면 이 단계는 스킵하고 바로 다음 단계로 진행. 액션포인트 상태 모델은 `TODO`/`DONE`/`FAILED`/`ROUTINE`/`REMOVED` 5분류로 확정(05번 참고). 현재 상태와 무관하게 4개 결정 모두 선택 가능(전이 제한 없음). 리뷰 화면과 대시보드는 `TODO`·`FAILED`·`ROUTINE` 상태를 모두 조회 대상으로 포함해야 함(복수 상태 필터, 구현 시 상세 결정) |

## FT-07. 최종 확인 및 KPT·액션포인트 수정

| 항목 | 내용 |
| --- | --- |
| 설명 | 모든 질문에 답변 후 누적된 K/P/T 전체를 검토하는 화면. 사용자가 직접 항목을 추가·수정·삭제할 수 있음. 화면 진입 시 확정된 Try 목록을 기반으로 AI가 액션포인트 후보를 생성해 함께 노출하고("액션포인트 작성" 단계), 기존 FAILED·ROUTINE 액션포인트도 이 목록에 이월해 content 수정 또는 철회(삭제)만 가능하게 한다(status는 여기서 변경 불가 — 상태 변경은 FT-04 리뷰에서만). 사용자는 신규 후보를 수정·삭제하거나 직접 새 항목을 추가할 수 있음 |
| 트리거 | 마지막 질문 답변 제출 |
| 관련 화면 | S-06 회고작성 - 최종확인 |
| 관련 API | `POST /retrospects/action-points/generate` (액션포인트 후보 생성), 이후 편집은 클라이언트 로컬 상태(저장 전까지 서버 호출 없음) |
| 비고 | 이 화면에서 편집한 K/P/T·액션포인트 내용이 저장 시점의 최종본이 됨. 신규 후보와 기존 항목 간 중복 체크는 하지 않음. 사용자 직접 추가 항목은 존댓말·표준어 보정 없이 입력 그대로 저장 |
