## 2. 문제 정의

혼자 회고를 작성하면 타인의 피드백이 없어 지속하기 어렵다. 무엇을 써야 할지 막막해서 빈 화면 앞에서 멈추는 경우가 많고, 써놓은 회고가 쌓여도 스스로 그 안에서 패턴이나 성장 신호를 읽어내기 어렵다.

## 3. 핵심 가치

**Git 활동 기반으로 AI 도움을 받아 편하게 회고를 작성하고, 회고마다 AI 분석·피드백을 받아 성장한다.**

- Git 활동을 자동 수집해 "오늘 무엇을 했는지"부터 정리해주는 부담을 덜어준다.
- AI가 활동 기반 질문을 던지고, 사용자는 질문에 답하는 방식으로 회고를 완성한다 (백지 공포 해소).
- 저장 시점에 액션포인트를 즉시 뽑아주고, 이후 비동기로 요약·인사이트를 분석해, 리스트로 쉽게 찾고 AI를 통한 피드백으로 성장한다.

## 4. 타겟 사용자

GitHub 활동은 꾸준하지만 혼자 회고를 지속하기 어려웠던 개인 개발자 (주니어~미드레벨). 팀/조직이 아닌 개인 성장 기록이 목적.

## 5. 비범위 (Out of Scope)

- 회고 수정 기능 (삭제만 가능, [[03-user-flow|유저플로우]] 참고)
- 팀/조직 단위 협업 기능
- 프로젝트 단위 목표를 해결하는 기능

## 6. 초기 단계 성공 기준

- 사용자가 "회고 작성 → AI 피드백 확인"의 루프를 부담 없이 반복할 수 있는가 (완주율)
- 초기 데이터가 적은 사용자도 막히지 않고 첫 회고를 완료할 수 있는가 (온보딩/가이드 유효성)
